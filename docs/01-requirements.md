# 01. 要件定義

## 1. プロダクト定義

| 項目 | 内容 |
|---|---|
| 正式名称（仮） | PreMeet（プリミート） |
| 一言説明 | 会社URLを入れるだけで商談前リサーチが1枚になるAIツール |
| 対象ユーザー | BtoB営業（法人営業・インサイドセールス）、士業、フリーランス受託者 |
| 利用シーン | 初回商談の前日〜直前、テレアポリスト作成時 |
| 提供形態 | Webアプリ（ログイン任意）。ネイティブアプリは作らない |

## 2. 解決する課題

営業パーソンは商談前に相手企業を調べるが、実態は以下のいずれかになっている。

- 時間がなく、会社概要をざっと見るだけで商談に臨む
- 調べてはいるが、「事業内容の把握」止まりで**仮説構築まで至らない**
- 調べ方が属人化しており、チームで品質が揃わない

本プロダクトは「調べる」ではなく **「仮説を持った状態にする」** ところまでを1クリックで届ける。

## 3. スコープ

### 3.1 やること（MVP）

- 企業URL または 企業名を入力
- 公開情報の自動収集（自社サイト / 採用ページ / ニュース検索）
- リサーチシートの生成（構造化JSON → 画面表示）
- 結果ページの共有リンク発行
- クレジット購入（Stripe 一回払い）
- 無料利用のレート制限

### 3.2 やらないこと（MVPスコープ外）

明示的に**やらない**。ここを守らないと完成しない。

- ログイン必須化（初回は匿名で使えること）
- CRM連携（Salesforce / HubSpot）
- チーム機能・権限管理
- 個人（担当者個人）に関する情報の収集・生成 → **法務上の理由。docs/07 参照**
- メール文面の自動送信
- リスト一括処理（バッチ）
- モバイルアプリ

### 3.3 将来的に検討（Phase 2以降）

- CSV一括投入（クレジット大量消費導線）
- 業界別テンプレート（IT / 製造 / 医療 / 建設）
- 生成履歴の保存とマイページ
- チーム共有ワークスペース

## 4. ユーザーストーリー

| ID | ストーリー | 優先度 |
|---|---|---|
| US-01 | 営業として、会社URLを入れるだけで相手企業の概要を把握したい | 必須 |
| US-02 | 営業として、相手企業が今抱えていそうな課題の仮説がほしい | 必須 |
| US-03 | 営業として、商談で使えるヒアリング質問を用意したい | 必須 |
| US-04 | 営業として、想定される断り文句と返し方を知っておきたい | 必須 |
| US-05 | 営業として、結果を上長やチームに共有したい | 高 |
| US-06 | 営業として、結果をPDFで持ち歩きたい | 中 |
| US-07 | 営業として、自社サービス情報を入れて「自社目線の切り口」を出したい | 高 |
| US-08 | 初見ユーザーとして、登録せずにまず試したい | 必須 |

> US-07 が転換率の鍵。自社サービス情報を保存させると、再訪動機とログイン動機が同時に生まれる。

## 5. 出力仕様（リサーチシートの構成）

| # | セクション | 無料 | 有料 | 生成元 |
|---|---|:--:|:--:|---|
| 1 | 企業サマリー（事業・規模・拠点） | ○ | ○ | 自社サイト |
| 2 | 提供サービス・主要顧客 | ○ | ○ | 自社サイト |
| 3 | 直近の動き（ニュース・リリース） | ○ | ○ | Web検索 |
| 4 | 採用状況から見た注力領域 | ○ | ○ | 採用ページ / 求人検索 |
| 5 | **想定課題 仮説 3つ**（根拠付き） | × | ○ | AI推定 |
| 6 | **刺さる切り口 3案** | × | ○ | AI推定 + 自社情報 |
| 7 | **ヒアリング質問 10問** | × | ○ | AI生成 |
| 8 | **想定反論と返し 5組** | × | ○ | AI生成 |

無料版は 1〜4（=事実の要約）まで。
**5〜8（=仮説と武器）が有料**。ここが価値の本体であり、課金理由として最も明快。

## 6. 非機能要件

| 項目 | 要件 |
|---|---|
| 生成時間 | 90秒以内（進捗をストリーミング表示し体感を短縮） |
| 同時実行 | Queue経由。1ユーザー同時1件まで |
| 可用性 | ベストエフォート。SLAは掲げない |
| 原価上限 | 1生成あたり $0.15 を超えないこと（超過時はモデル降格 or 中断） |
| サポート | FAQ + 問い合わせフォームのみ。チャットサポートは置かない |
| ログ | 入力URL・生成結果は保存。**個人情報は保存しない** |

## 7. 成功指標（KPI）

Phase 1 の判定基準。これを下回ったら方向転換する。

| 指標 | 目標 |
|---|---|
| 無料生成 → 結果閲覧完了率 | 70%以上 |
| 結果閲覧 → クレジット購入率 | 2%以上 |
| 再訪率（7日以内） | 15%以上 |
| 1生成あたり平均原価 | $0.08 以下 |

> 購入率2%は控えめな数字だが、無料層の広告・アフィリで補完する前提。
> ここが0.5%を切る場合、有料セクションの設計を見直す。
