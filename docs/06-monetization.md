# 06. 収益設計・原価防衛

## 1. 料金体系

**サブスクリプションは採用しない。** クレジット買い切り方式。

| パック | 価格 | クレジット | 単価 | 有効期限 |
|---|---|---|---|---|
| スターター | 980円 | 10 | 98円 | 6ヶ月 |
| スタンダード | 2,980円 | 40 | 74円 | 6ヶ月 |
| プロ | 7,980円 | 150 | 53円 | 6ヶ月 |

- 1レポート = 1クレジット
- 有効期限6ヶ月は**法務上の理由**（docs/07 参照）。UI上で購入前に必ず明示する
- 初回ログイン時に **3クレジットを無料付与**（`reason: 'bonus'`）→ 有料版の価値を体験させる

### 無料層

| 対象 | 回数 | 内容 |
|---|---|---|
| 匿名 | 1日2回 | セクション1〜4のみ |
| ログイン済 | 1日3回 | セクション1〜4のみ |

## 2. 原価構造

1レポートあたりの想定原価（**概算。実測で必ず更新すること**）

| 項目 | 無料 | 有料 |
|---|---|---|
| ページ取得（Workers） | ほぼ0 | ほぼ0 |
| Web検索API | $0.005 | $0.005 |
| Stage1（入力大・安価モデル） | $0.01〜0.02 | $0.01〜0.02 |
| Stage2（入力小・上位モデル） | — | $0.02〜0.05 |
| Supabase / R2 | ほぼ0 | ほぼ0 |
| **合計** | **$0.02前後** | **$0.04〜0.08** |

有料単価74円（≒$0.5）に対し原価$0.08なら **粗利率85%程度**。健全。

> ただしこれは「収集ページ数10・本文8,000文字トリム」を守った場合の数字。
> 上限を外すと簡単に3倍になる。**上限はコードで強制すること。**

## 3. 原価防衛の実装チェックリスト

実装時に以下がすべて入っていることを確認する。1つでも欠けると原価が暴走する。

- [ ] 収集ページ数の上限（10ページ）をコードで強制
- [ ] 1ページあたりの本文トリム（8,000文字）
- [ ] 同一ドメイン7日キャッシュ（`companies.facts_generated_at`）
- [ ] 匿名レート制限（Cookie）
- [ ] **IPレート制限**（Cookie削除による回避を防ぐ）
- [ ] 無料層は Stage2 を実行しない
- [ ] AIリトライは最大1回
- [ ] Queue の `max_retries = 2` + Dead Letter Queue
- [ ] 1日の総生成数のグローバル上限（サーキットブレーカー）
- [ ] `cost_usd` を全レポートに記録し、日次で確認できるようにする

### サーキットブレーカー

```typescript
// 想定外のトラフィック/攻撃で原価が暴走するのを止める最後の砦
const DAILY_GLOBAL_LIMIT = 500;   // 全ユーザー合計の1日生成上限

const count = await env.RATE_LIMIT.get(`global:${today}`);
if (Number(count ?? 0) >= DAILY_GLOBAL_LIMIT) {
  // 有料ユーザーのみ通す。無料は一時停止
  if (tier === 'free') throw new AppError('RATE_LIMITED');
}
```

**個人開発の無人運用では、この1機能があるかどうかで最悪の損失額が決まる。**
必ずMVPに入れること。

## 4. 無料層の収益化

### アフィリエイト

結果ページ下部・サイドに配置。

| 商材 | ASP候補 | 相性 |
|---|---|---|
| 営業支援ツール（SFA/CRM） | ASP各社の法人向け案件 | 高 |

- PVが月3万を超えたら検討

## 5. 集客（ここが唯一の「実作業」）

無人運用でも集客だけは自動化できない。以下を継続する前提で計画すること。

### SEO記事（最優先）

狙うキーワード群

| キーワード | 意図 | 記事の型 |
|---|---|---|
| 商談 事前準備 | 手法を知りたい | ハウツー + ツール導線 |
| 営業 企業研究 やり方 | 同上 | チェックリスト配布 |
| 初回商談 質問 例 | すぐ使いたい | 質問リスト50選 |
| ヒアリングシート テンプレート | 資料が欲しい | テンプレ配布 → ツール誘導 |
| 商談 断り文句 切り返し | 対処法 | 事例集 |
| BtoB 課題 仮説 立て方 | 深い層 | 解説記事 |

記事末尾に「この記事の内容を自動でやるツール」として自然に導線を置く。

### プロダクト内の拡散導線

- 共有リンクのOGP画像に「PreMeetで生成」を入れる
- PDF出力のフッターにURL
- 結果ページは `is_public` にすればSEO対象になる（**要検討**：企業名で検索流入する反面、
  相手企業から見られるリスクがある。デフォルトは非公開にすべき）

## 6. 特定商取引法・利用規約

以下のページはリリース前に必須。

- 特定商取引法に基づく表記（**クレジット販売＝有償提供なので必須**）
  - 事業者名（Nexeed Lab / 屋号と代表者名）、所在地、連絡先、価格、支払方法、
    提供時期、**返品・キャンセルの可否**
- 利用規約（生成結果の正確性を保証しない旨、商用利用の範囲、禁止事項）
- プライバシーポリシー

> 所在地・電話番号の表記が必要になる点は、自宅住所の公開を伴うため事前に方針を決めること。
> バーチャルオフィスの利用を検討する余地がある。
