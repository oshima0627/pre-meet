# 08. 開発ロードマップ

## 原則

**Phase 0 を飛ばさない。** ここで精度が出なければ、以降を作っても価値はゼロ。
UI・決済・SEOはすべて Phase 0 の合格が前提。

---

## Phase 0：精度検証（3〜5日）

UIを作らない。CLIスクリプトのみ。

### やること

1. `scripts/verify.ts` を書く
   - 引数：企業URL
   - 処理：ページ収集 → Stage1 → Stage2 → JSON をコンソールとファイルに出力
2. 業種の異なる **10社**で実行
   - IT / 製造 / 小売 / 建設 / 医療 / 士業 / 飲食 / 物流 / 不動産 / 教育
   - 大企業と中小を混ぜる（中小は情報が薄く、ここで破綻しやすい）
3. docs/05 の検証表で手採点

### 合格基準

| 項目 | 基準 |
|---|---|
| 事実の混入エラー | 10社中0件 |
| 妥当な課題仮説 | 10社中7社で「2件以上」が妥当 |
| 汎用出力（どの企業にも当てはまる内容） | 10社中2社以下 |
| 情報が薄い企業での挙動 | 無理に生成せず THIN_CONTENT で止まる |
| 1件あたり原価 | $0.10以下 |

**不合格だった場合の打ち手**
- 収集ページの選び方を見直す（採用ページ・事例ページの取得率を上げる）
- Stage2に「業種」を明示的に渡し、業種別の観点を与える
- それでもダメなら **プロダクトを中止**して別案に切り替える

- CSV一括投入（クレジット大量消費導線 = 客単価が跳ねる）

> 拡散力が非常に高い。Phase 4の第一候補。

---

## 中止判断の基準

以下に該当したら、固執せず撤退する。

- Phase 0 の合格基準を、プロンプト改善2周しても満たせない
- Phase 1 リリース後3ヶ月で、購入者が累計10人未満かつ再訪率5%未満
- 1レポート原価が平均$0.15を超え、下げられない

**個人開発の最大のコストは時間であり、撤退基準を先に決めておくことが最大の防御。**
